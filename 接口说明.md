# 后台模块相关接口

## 一、菜单管理

### **1.菜单添加** 

**请求URL：** 

- /api/menuadd

请求方式：

- POST 

参数：

| 参数名 | 说明                                       |
| :----- | ------------------------------------------ |
| pid    | 上级分类编号，  pid字段必须添加，0表示顶级 |
| title  | 标题                                       |
| icon   | 图标                                       |
| type   | 类型1目录2菜单                             |
| url    | 菜单地址                                   |
| status | 状态1正常2禁用                             |

### **2.菜单列表** 

请求URL：

- /api/menulist
- /api/menulist?istree=1   **获取树形菜单数据，重要！！！**

请求方式：

- GET

### 3.菜单获取（一条）

请求URL：

- /api/menuinfo

请求方式：

- GET

参数：

| 参数名 | 说明           |
| :----- | -------------- |
| id    | 菜单编号   |

可以根据指定的编号获取到一条对应的菜单信息

### 4.菜单修改

请求URL：

- /api/menuedit

请求方式：

- POST 

参数：

| 参数名 | 说明           |
| :----- | -------------- |
| id     | 编号，必填项   |
| pid    | 上级分类编号   |
| title  | 标题           |
| icon   | 图标           |
| type   | 类型1目录2菜单 |
| url    | 菜单地址       |
| status | 状态1正常2禁用 |

### 5.菜单删除

请求URL：

- /api/menudelete

请求方式：

- POST 

参数：

| 参数名 | 说明           |
| :----- | -------------- |
| id    | 菜单编号   |

可以根据指定的菜单编号进行删除  **注意：一级目录下有菜单 就不可以删除**

请求URL：

- /api/getcategoods

请求方式：

- GET

参数：

| 参数名 | 说明             |
| :----- | ---------------- |
| fid     | 分类编号，必填项， 传入fid 则获取该一级分类下的所有商品 |
| sid    | 分类编号，必填项， 传入sid 则获取该二级分类下的所有商品 |
| page    | 页码数，默认1 |
| size    | 每页数量，默认10 |

**fid 和 sid 二者必须要传入一个。 如果都传入了，那么只有fid有效。** 返回的total表示所有数据条数
